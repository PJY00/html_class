<form>

form 태그를 Tab으로 자동완성 하면 action이라는 속성이 나옴.
action 속성은 해당 입력값을 보내는 주소임.
나중에 쓰이기에 알아둬야 하지만 지금은 그냥 입력값을 보낼 주소를 action에 넣는다는 사실만 기억하기.
그리고 method 속성이 존재함.
java를 들었던 사람들은 기억하겠지만, GET과 POST 값을 저기에 넣음.
입력 값을 어떻게 보낼지를 정함.
Get은 주소에, post는 숨겨서
얘도 나중에 자세히 설명하겠음.
그래서 보통 get은 검색, 조회처럼 단순 요청 및 민감정보가 담기지 않은 것에 쓰이고,
post는 로그인처럼 민감정보를 담고 있을 때 사용

        
            <form action="보낼 주소" method="GET 또는 POST">
                입력 요소들...
            </form>

<input>과 <label>

이 둘은 보통 쌍으로 쓰임. input은 정말 input 값을 입력하는 부분이고, label은 이에 대한 설명부라고 볼 수 있음. 

        
            <label for="ID">아이디</label>
            <input type="text" name="ID" id="ID">

그리고 이 input에서의 type이 우리가 입력할 타입들을 결정함.
대표적으로 text, password, date, checkbox, radio가 있음.
여기서 name 속성이 필요한데, radio나 checkbox의 경우 name이 같으면 같은 묶음으로 인식.
그래서 radio의 경우 name이 같으면 하나만 선택할 수 있고,
checkbox의 경우 name이 같아도 여러 개 선택 가능하지만 서버에서 배열로 인식함.
input 값마다 name이 있어야 서버가 이 값이 무슨 값인지 구분 가능.

<select>와 <textarea>

        
            <label for="city">거주 지역:</label>
            <select id="city" name="city">
                <option value="seoul">서울</option>
                <option value="busan">경기</option>
                <option value="daegu"<충청>/option>
                <option value="etc">기타</option>
            </select>

select는 위처럼 value에 서버에 보낼 값을 넣음.
input에서 radio랑 checkbox도 그럼.
다만 text에서는 미리 입력되어있을 값을 value에 넣음.

textarea는 폼 전송시 서버에 그대로 넘어감.

<button>

form 안의 버튼은 기본 타입이 submit, 제출용 버튼임.
다른 타입으로는 reset(초기화), button(아무 기능 없는 일반 버튼)이 있음
폼 외부에서는 button이 기본 타입. 

        
            <button type="submit">제출하기</button>

추가 속성

placeholder과 required가 있는데 얘는 직접 해보는 게 좋음.
placeholder은 힌트, required은 필수 입력임.
그리고 maxlength와 minlength 딱봐도 최대 글자수, 최소 글자수인 속성도 존재 

        
            <label for="id">아이디:</label>
            <input type="text" id="id" name="id" placeholder="아이디를 입력하세요.(3~10글자)" maxlength="10" minlength="3" required>

input tag로 버튼 만들기

      
        <input type="submit" value="제출">
